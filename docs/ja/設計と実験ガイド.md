# 双曲 SimCSE 設計と実験ガイド

本書は、実装済みコードベース `hypsimcse` の設計思想・数式・各モジュールの役割・
実験の回し方を日本語でまとめたものです。仕様書 PDF
（`~/Downloads/hyperbolic_simcse_experiment_spec.md.pdf`）と対応しています。

---

## 1. 全体像

標準的な SimCSE（超球面 / vMF 上の対照学習）の**台多様体だけ**を
双曲空間（Lorentz 模型）へ差し替えます。エンコーダ出力より後段（射影・スコア・損失）
のみを変更する「クリーンな ablation」を狙い、拡張戦略やハードネガティブ設計といった
交絡要因を増やさずに幾何の効果だけを切り出します。

学習信号は 3 系統：

| トラック | 正例の作り方 | 役割 |
|----------|-------------|------|
| **純グラフ**（`graph`） | synset を直接ベクトル化し hypernymy の辺を正例 | 幾何の効果を孤立（Nickel–Kiela の複製 + スコア ablation） |
| **エンコーダ + 教師あり**（`encoder`） | gloss を BERT で符号化、木の辺 (下位, 上位) を正例 | 主トラック。未知 synset への帰納的汎化が測れる |
| **dropout**（`dropout`） | 同一 gloss を 2 回符号化（dropout ノイズが正例） | 下限ベースライン（教師なし SimCSE） |

## 2. 幾何コア（`src/hypsimcse/geometry/`）

### 2.1 Lorentz 模型（`lorentz.py`）— 付録 A の式に忠実

- 双曲面：`H^d_c = { x ∈ R^{d+1} : ⟨x,x⟩_L = -1/c, x_0 > 0 }`
- Lorentz 内積：`⟨x,y⟩_L = -x_0 y_0 + Σ_{i≥1} x_i y_i`
- 距離：`d_H(x,y) = (1/√c) · arccosh(-c⟨x,y⟩_L)`。`c=1` では `⟨x,y⟩_L = -cosh d_H`。
- 原点 `o = (1/√c, 0, …, 0)`。
- **空間成分のみ保持**（`d` 個）し、時間成分は制約から `x_0 = √(1/c + ‖x_space‖²)` で復元（MERU と同じ流儀）。埋め込み次元 `d` ＝ 空間成分の数。

主な関数：

- `lorentz_inner(x, y)` … Lorentz 内積
- `expmap0(v_space, c, max_norm)` … 原点での指数写像（接ベクトル → 双曲面点）。`max_norm` で接ベクトルのノルムを上限クランプしオーバーフローを防止。
- `dist(x, y, c)` … 双曲距離（arccosh の引数を `[1+ε, ·]` にクランプ）

> 実装ノート：float32 では指数写像の丸め誤差で `⟨z,z⟩_L` が厳密に `-1` にならず、
> 自己距離が最大 ~4e-3 程度になります。テストの許容誤差はこれを踏まえて設定しています（バグではなく float32 の限界）。

### 2.2 スコア関数（`scores.py`）— 主たる実験因子

InfoNCE の類似度 `sim` を差し替える 4 種。`pairwise_scores(u, v, label, c)` が
`(N, M)` の類似度行列を返します。

| ラベル | スコア | 由来 |
|--------|--------|------|
| `EUC` | `cos(u,v)`（L2 正規化 + 内積） | 標準 SimCSE（超球面/vMF） |
| `HYP-dist` | `-d_H(u,v)` | 既存双曲対照学習の主流（MERU 既定） |
| `HYP-dist2` | `-d_H(u,v)²` | Riemannian Gaussian 近似 |
| `HYP-inner` | `c·⟨u,v⟩_L`（`c=1` で `-cosh d_H`） | **本研究の理論予言（負 Lorentz 内積）** |

`EUC` のみ入力は接ベクトル（`(N,d)`）、それ以外は双曲面点（`(N,d+1)`）を受け取ります。

## 3. 損失（`src/hypsimcse/losses/infonce.py`）

全条件で共通の InfoNCE：

```
L = -log( exp(sim(z_i, z_i⁺)/τ) / Σ_j exp(sim(z_i, z_j)/τ) )
```

差分は `sim`（スコア）のみ。バッチ内の他行をネガティブとして扱います（NT-Xent 形式）。

- `info_nce(z, z_pos, label, tau, c)` … 上式。行 i の正例は `z_pos[i]`。
- `root_anchor_reg(z, c)` … **H4 用**。原点からの平均双曲距離を「負」で返し、
  最小化すると点群が根から外側へ散らばる（uniformity 項の代替）。
- `total_loss(...)` … InfoNCE + `anchor_weight × root_anchor_reg`。内訳を dict で返す。

## 4. データ（`src/hypsimcse/data/`）

### 4.1 `wordnet.py`
WordNet 3.0 名詞 synset をノード、hypernymy の**推移閉包**を辺とします（N–K と同一設定）。
`gloss_text()` は「先頭 3 lemma ＋ ` : ` ＋ 定義文」を小文字化・空白正規化して返す固定前処理
（文長・lemma バイアスの統制）。

### 4.2 `splits.py`
- `reconstruction_split` … 全辺で学習・評価（埋め込み品質の上限）
- `link_prediction_split` … 辺を train/valid/test に分割（未観測辺の予測）
- `inductive_split` … synset を分割し、hold-out synset に触れる辺を eval に回す（transductive な N–K では不可能な設定）
- `negatives_for` … 真の上位語を除いてネガティブをサンプリング（順位づけ指標用）

### 4.3 `graph_stats.py`（データ側の事前計量）
- `depths` … 根からの最短ホップ深さ（H4 のノルム-深さ相関の基準）
- `branching_factors` / `depth_distribution` … 分岐数・深さ分布
- `gromov_delta` … サンプリングによる 4 点 Gromov δ 双曲性（0 に近いほど木的 ＝ 双曲が効く前提）

> WordNet 名詞は多重継承の DAG なので、「子の深さ ≥ 親の深さ」は必ずしも成り立ちません
> （別の親経由で根に近い場合がある）。テストは BFS が保証する `depth[子] ≤ depth[親]+1` を検証します。

## 5. モデル（`src/hypsimcse/models/`）

### 5.1 共有 lift ヘッド（`lift.py`）
`HyperbolicHead(in_dim, out_dim, geometry, c, learnable_c, max_tangent_norm)`。
`geometry="HYP"` なら線形射影 → 指数写像で双曲面へ（`(B, d+1)`）。
`geometry="EUC"` なら接ベクトルをそのまま返す（cosine はスコア側で処理）。
`learnable_c=True` で曲率を `softplus(raw_c)` として学習可能に。`.curvature()` で現在値取得。

### 5.2 純グラフ埋め込み（`graph_embedding.py`）
`nn.Embedding` で synset ごとに 1 接ベクトルを保持し、同じ lift ロジックで多様体へ。
エンコーダなし。幾何の効果を孤立させます。

### 5.3 文エンコーダ（`encoder_model.py`）
`SentenceEncoder.build(model_name, out_dim, geometry, ...)` で
BERT/RoBERTa（`AutoModel`）＋ `HyperbolicHead` を構築し、tokenizer も返します。
pooling は `cls` / `mean`。`encode_texts(...)` は評価時の no-grad バッチ符号化ヘルパ。

## 6. 学習（`src/hypsimcse/training/`）

- `config.py` … `ExperimentConfig`（全ハイパラを保持、YAML と相互変換）
- `seeding.py` / `device.py` … 乱数シード固定、device 自動選択（cuda→mps→cpu）
- `trainer.py` … config 駆動の学習ループ。1 つのループが 3 トラック全てを扱い、
  正例の作り方とモデルだけが分岐する。

**数値安定化スタック（MERU の失敗を踏まえた対策、どれが効いたかは `train_logs` に記録）:**

- arccosh / sqrt の引数クランプ、Lorentz ノルムクランプ（`geometry/lorentz.py`）
- 接ベクトルのノルム上限スケーリング `max_tangent_norm`（`lift.py`）
- 勾配クリッピング `grad_clip`、非有限損失のステップスキップ（`skipped` を記録）
- 最適化：接空間経由の lift なのでエンコーダ・射影 `W` は通常の **AdamW** で可。
  直接オン多様体パラメータを持つ場合のみ **Riemannian Adam**（geoopt）を `optimizer: radam` で比較。
- `HYP-inner` は不安定化しやすいので学習率を小さめに設定可能。

`dropout` トラックでは正例を**2 回目の forward**で作り、独立した dropout マスクで
別視点を得ます（同一ベクトルを使い回すと自明に一致してしまうため）。

## 7. 評価（`src/hypsimcse/eval/`）

| モジュール | 指標 | 目的 |
|-----------|------|------|
| `reconstruction.py` | MAP、mean rank | 幾何の純粋な階層保存力（主指標） |
| `link_prediction.py` | MAP、AUC | 未観測の上位下位関係を当てる汎化 |
| `inductive.py` | 未知 synset の reconstruction MAP | エンコーダ版の固有貢献 |
| `sts.py` | STS-B Spearman | 一般意味を壊していないかの保険 |
| `fisher.py` | 動径/接固有値、条件数 | **H3**：温度の関数としての異方的崩壊 |
| `hierarchy.py` | ノルム vs 深さ相関、埋め込みの δ | **H4**：根アンカー型の specificity 信号 |
| `dual_coords.py` | Einstein midpoint との一致（gap） | 理論と実装の橋（`η=∇ψ`） |

- **Fisher 崩壊診断**：埋め込みを原点の接座標へ写して経験共分散（＝バッチ双曲 vMF の Fisher）を推定し、
  平均方向に沿う**動径分散**と直交する**接分散**に分解、条件数 `max固有値/min固有値` を報告。
- **双対座標の同一視**：ソフトマックス重み付き Lorentz 平均（モデルが暗に使う平均）が
  Einstein midpoint（`η=∇ψ(θ)`）に一致するかを距離で測り、理論↔実装を実証。

## 8. 実験（`experiments/`）と必須プロット

`experiments/configs/E1..E6.yaml` が仕様書の実験マトリクスに 1:1 対応。

- `run.py` … 1 設定を train + eval して JSON 結果を出力。`--full-hierarchy` で
  ノルム/深さの生配列も保存（図 2 用）。
- `run_matrix.py` … 次元 / スコア / 温度 / シードを展開し、セルごとに JSON を書き出す（E1〜E3）。
- `plots.py` … 必須 3 プロット
  1. **metric vs 次元**（H1）：`plot_metric_vs_dim`
  2. **ノルム vs 深さ**（H4）：`plot_norm_vs_depth`
  3. **Fisher 条件数 vs 温度 τ**（H3）：`plot_condition_vs_tau`

各セルは 3 シードで実行し、主表は平均 ± 標準偏差で報告します。
エンコーダ・最適化・バッチサイズ・エポックは条件間で固定します。

## 9. 想定される失敗モードと対処（仕様書 §7）

- **`HYP-inner` の最適化不安定** → §6 の安定化を段階適用し、`train_logs` の `skipped`/`part_*` で
  どれが効いたかを記録（これ自体が知見）。
- **双曲が勝たない** → データの δ が大きい可能性。`graph_stats` の事前計量で予測し負の結果として整理。
- **STS が劣化** → 一般意味と階層特化のトレードオフとして報告。
- **gloss 文の質依存** → 前処理を `gloss_text` に固定し文長バイアスを統制。

## 10. 計算環境の注意

本マシンは macOS + Apple MPS（CUDA なし）です。主トラック（BERT-base × 全 8.2 万 synset ×
3 シード）は MPS ではフル実行が非常に重いため、まずは**純グラフ版**で H1/H2/H3 を出すのが実際的です。
`--max-synsets` で規模を縮小して動作確認できます。GPU 環境が用意でき次第、
同じコードでフル規模へスケールします。

## 11. 既知の簡略化（バグではなく v1 の割り切り）

- reconstruction の `distortion`（平均歪み）は指標関数側では `nan` を返します。
  グラフ最短路計量を呼び出し側で渡した時に埋める設計で、主指標は MAP / mean rank です。
- δ 双曲性・双対座標 gap は計算量のためサンプリング（サンプル数は引数で調整可）。
- Riemannian Adam は配線済みですが `optimizer: radam` の時のみ動きます。
